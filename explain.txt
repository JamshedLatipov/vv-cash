Yes, I am adding the `Cash-Authorization` header implicitly.

Here is how Dependency Injection works in this application:
In `App.axaml.cs`:
```csharp
services.AddHttpClient("DefaultClient").AddHttpMessageHandler<AuthHeaderHandler>();
services.AddTransient(sp => sp.GetRequiredService<IHttpClientFactory>().CreateClient("DefaultClient"));
```
This registers a default transient `HttpClient` instance for the entire application that automatically has `AuthHeaderHandler` attached to it.

Any service that gets an `HttpClient` injected into its constructor, like my `CategoryService`:
```csharp
public CategoryService(HttpClient httpClient, ISettingsService settingsService)
```
Will automatically receive this default configured client.

The `AuthHeaderHandler.cs` automatically intercepts every outbound request and appends:
```csharp
request.Headers.TryAddWithoutValidation("Cash-Authorization", token);
request.Headers.Authorization = new AuthenticationHeaderValue("Bearer", authToken);
```

So yes, both `Cash-Authorization` and `Authorization: Bearer` are automatically being appended to all HTTP requests made by `CategoryService`. I didn't have to manually add it to the code inside `CategoryService.cs`.
